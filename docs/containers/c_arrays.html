C Arrays

These are the simplest type of container. So simple in fact that they don't really count as one, but they are used everywhere and form the foundation of most containers.

An array in C is simply a pointer to a fixed size region of memory. Pointers will be explained later, but for now a pointer and an array can be thought of as the same.

We have already seen an array before - argv from main. argv is an array of char* and char* is an array of characters. argv can actually be written in two equivalent forms:

char** argv
char* argv[]

These two only differ when we first define our array. Here are two definitions on an integer array to explain the differences:

int my_array[] =
{
    ... // comma separated elements
    // size is filled out automatically from how many elements we have
    // the array is initialised with these elements when the program starts
};

int my_array[5]; // define an array of 5 ints

int* my_array; // define an empty array (must be allocated manually)

There are some rules to how arrays are allocated. An array defined using square brackets [] can be defined in two places - inside a block or outside of any block. When defined inside a block, this array can take a variable as it's length:

int main()
{
    int length = 42;
    int my_array[length]; // 42 ints
    length++; // length is now 43, array still contains 42 ints
}

When defined outside of a block, the array can only take a literal value (not a variable):

int my_array[42]; // this is okay

int length = 42;
int my_array[length]; // this is not okay

An array defined using a pointer * has no defined length, and starts without any data. These arrays need to be initialised. We can either allocate the memory ourselves or point this array to an existing array's data.

int main()
{
    int my_array[42];

    int* my_array_2 = my_array; // has the same data as my_array
    // this is not a copy of the original array, it IS the original array

    int* my_array_3 = my_array_2; // same applies here

    // malloc takes a number of bytes to allocate, so we need to multiply our length by the size of one element
    int* my_array_4 = malloc(42 * sizeof(int)); // this allocates a new array of 42 ints
    // anything allocated using malloc must be free'd later
    free(my_array_4);
}

Arrays formed with malloc can be resized, leading to variable length arrays (dynamic arrays or sometimes called vectors):

int main()
{
    int* my_array = malloc(42 * sizeof(int));
    my_array = realloc(my_array, 52 * sizeof(int));
    // error handling for realloc not included for simplicity

    free(my_array);
}

realloc can return NULL, so it's result should be stored in a temporary variable and checked. This is because if realloc fails, it does not free the original array, and by setting it to NULL, we can no longer free it ourselves, so the array will remain in memory until the program exits. This is called a memory leak.

Indexing an element from an array is easy - to index element n from an array my_array, use my_array[n]. The first element has index 0. The reason for this will be explained later.
