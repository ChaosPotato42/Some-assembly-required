Makefiles

This chapter is a GNU make crash course. It will explain how to create a build system for almost any C/C++ project.

GNU make works by specifying targets. These targets 'produce' a file from a set of arguments - for example:

%.o: %.c
    gcc -c $< -o $@

%.o specifies a file with the .o extension. %.c specifies a file with the same name, but a .c extension. $< is the first argument (in this case %.c) and %@ is the target (%.o). We can also use $^ which specifies all arguments

In other words, this target means 'to create a file X.o, we need X.c, and to run gcc -c X.c -o X.o'

The -c flag in gcc tells it to output an object file (.o) and -o tells it the name of the file to output.

Another important concept in GNU make is variables. Variables can be defined, redefined and appended. To define or redefine a variable 'VAR' use VAR := ....

Note: the first (or only) definition of a variable can also be written as VAR = ....

To append a variable, either use VAR := $(VAR) ..., or VAR += ....

With this information, we can use variables in our targets:

TARGET = main
OBJECTS = main.o

$(TARGET): $(OBJECTS)
    gcc $^ -o $@

%.o: %.c
    gcc -c $< -o $@

This makefile will compile main.c into main.o, and then produce main.

This becomes more powerful if we have a large project with many files, as we can add them to OBJECTS, or even use a file search to find all source files and compile them.

Compiler flags can also be defined easily inside makefiles. For gcc, the variable name CFLAGS is often used, and for c++, CXXFLAGS is often used. For the final stage where object files are linked into a program, LDFLAGS is used. Our simple examples don't currently require any flags, but they will be introduced as an when required.

It is also important to note that C and C++ source files both produce .o object files that can be linked together. This will be covered in more depth later, but the idea is that a project can utilise both languages.

Later we will build a makefile generator to configure large projects effortlessly, but for now, this is more than sufficient.
